Security Best Practices

Critical Security Warning

Your private key controls your funds. Anyone with access to your private key can steal your assets. Treat it like your bank password - but even more carefully!

‚úÖ DO - Security Best Practices

  • Use Test Accounts on Buildnet
    Only use accounts with test funds when experimenting on buildnet
  • Store Keys Securely
    Use encrypted storage or hardware wallets for production keys
  • Verify Website URLs
    Always check you're on the correct website before entering credentials
  • Use Environment Variables
    For automated deployments, use environment variables instead of config files
  • Regular Backups
    Keep secure backups of your keys in multiple safe locations

‚ùå DON'T - Common Mistakes

  • Never Share Your Private Key
    No legitimate service will ever ask for your private key
  • Don't Commit Keys to Git
    Always add wallet config files to .gitignore
  • Avoid Plain Text Storage
    Don't save keys in unencrypted text files or emails
  • Don't Use Production Keys for Testing
    Always use separate accounts for development and production
